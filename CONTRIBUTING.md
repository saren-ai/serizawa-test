### 2. Dispute a Trope Detection

If the app has misclassified a trope detection for a specific character, you can dispute it.

**Valid dispute reasons:**
- False positive: the trope was detected but doesn't apply to this portrayal
- Missed context: the detection ignores a clear subversion or narrative deconstruction
- Wrong severity: the assigned severity is inconsistent with the taxonomy definition

**Not valid:**
- "I like this character / this movie" â€” personal preference is not a dispute
- Disagreement with the trope taxonomy itself â€” that's a separate process (see Â§4)

**Via the app:** Character detail page â†’ Trope pill â†’ Dispute button  
**Via GitHub:** Open an Issue using the [Trope Dispute](.github/ISSUE_TEMPLATE/trope-dispute.md) template

Required fields: Character name + media, trope ID, dispute reason (min 50 chars), evidence.

Disputes resolved in your favor count toward Critic eligibility (see Â§5).

## What We Don't Accept

- Submissions defending yellowface, ethnic caricature, or harmful tropes as "products of their time" without critical engagement
- Personal attacks on actors, directors, or community members
- Proposals to remove characters from the Wall of Shame based on nostalgia
- Trope submissions targeting non-Japanese Asian representation (this framework is specifically Japanese / Nikkei â€” cross-ethnicity adaptation is a future roadmap item)

## Code of Conduct

This project exists to serve Japanese-American community members and scholars who have direct experience with the representations we analyze. Contributions must reflect that purpose.

Treat disagreements about representation with the seriousness they deserve. These aren't abstract debates â€” they're about how real people see themselves reflected (or erased) in media.
